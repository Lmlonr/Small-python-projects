# Small-projects
a)At Viessmann and Sodium, the primary programming languages used include Python, JavaScript, and Java.For testing purposes, Python is extensively used with tools like pytest, unittest, and Selenium.
Development environments vary depending on the team, but common IDEs include Visual Studio Code, PyCharm, and IntelliJ IDEA.
Version control is handled via Git, and repositories are hosted on platforms like GitHub and GitLab.
b)Both companies utilize modern web technologies for full-stack development.
On the frontend, they use technologies like React.js, HTML5, CSS3, and Bootstrap for responsive interfaces.On the backend, they employ Node.js, Java (Spring Boot), and Python (Django/Flask) depending on project requirements.RESTful APIs are used for communication between client and server components.
c)Viessmann and Sodium rely on both relational and non-relational databases.
Relational databases such as PostgreSQL and MySQL are commonly used for structured data, while MongoDB is used for NoSQL-based storage when flexibility is needed.
Database access is managed securely, and test environments often use anonymized or dummy data to protect sensitive information.
d)The network infrastructure includes secure VPN access, firewall protection, and role-based access control.
Development and staging environments are hosted on internal or cloud-based servers with appropriate security layers.The systems are segmented into production, staging, and development networks to avoid interference and ensure stability.
e)For testing, the companies use Postman for API validation, Selenium for browser automation, and JMeter for load testing.For CI/CD, tools like Jenkins, GitHub Actions, and Docker are implemented.
Documentation and communication tools include Confluence, Slack, and Microsoft Teams.
Version control is performed through Git, and some projects use Docker Compose for environment simulation.
f)Project management is carried out primarily through Jira, where tasks are tracked using Agile methodologies like Scrum and Kanban.Sprint planning, task assignment, bug tracking, and documentation are managed through Jira boards.
Time tracking tools and stand-up meetings are part of daily processes to ensure effective monitoring and collaboration.
